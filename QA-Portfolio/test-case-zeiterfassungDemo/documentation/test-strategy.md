
## Ziel
Ziel ist es, die Qualität und Stabilität der Zeiterfassungs-App sicherzustellen

## Testumfang 
Geteste werden:
-  Login
-  Benutzeranmeldung
-  Erstellen von Zeiteinträgen
-  Bearbeiten von Zeiteinträgen
-  Validierungen
-  Fehlermeldung

Nicht getestet werden:
-  Performance Tests
-  Sicherheitstests
-  Lasttests

## Testarten

Folgende Testarten werden durchgeführt:
-  Funktinonale Tests
-  Smoke Tests
-  Regessionstests
-  Negative Tests

## Testmethoden
Manuelle Tests
-  Testfälle ausführen
-  Fehler dokumentieren

Automatisierte Tests
-  C# + Playwright
-  Automatisierung wichtiger Abläufe

## Testumgenung

Browser:  Chrome
System:   Testumgebung lokal

## Qualitätskriterien

Die Anwendung gilt als aureichend getestet wenn:
-  Kritische Fehler behoben sind
-  Somke Test erfolgreich sind
-  Hauptfunktionen funktionieren
